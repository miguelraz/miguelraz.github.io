Kernel transformation

Input: kernel function K(r) K(r) K(r). Output "first integral" of K K K :

Kn(X)=∫01wnK(wX)dw \mathcal{K}_n(X) = \int_0^1 w^n K(wX) dw Kn​(X)=∫01​wnK(wX)dw

for X∈(0,∞)X \in (0,\infty)X∈(0,∞). Some small set of nnn values, to be determined, will be supplied at compile time.

KKK will be specified as a subtype of AbstractKernel.

  • For some kernel types, Kn\mathcal{K}_nKn​ is known analytically. (See Table I in Homer's paper.)

  • KKK may depend on some compile-time numeric parameters, specified as type parameters.

  • If the Kn\mathcal{K}_nKn​ integral is not known analytically, we may have to compute it numerically. This will be done at compile time by staged functions, with the numeric integration results used to compute the coefficients of a Chebyshev polynomial fit, which can then be compiled into an efficient polynomial approximation of Kn\mathcal{K}_nKn​. However, to compute a Chebyshev approximation for a function defined on (0,∞)(0,\infty)(0,∞), we will have to perform a coordinate transform from X→(0,1)X \to (0,1)X→(0,1), and the type of coordinate transformation will depend on how fast Kn\mathcal{K}_nKn​ decays asymptotically as X→∞X\to\inftyX→∞. This decay rate can be specified via the type of K.

The Kn\mathcal{K}_nKn​ function will be parameterized by a FirstIntegral{K,n} type parameterized by an AbstractKernel type K and an integer n.

Analytically known integrals:

immutable PowerLaw{p} <: PowerLawScaling{p,p,p} end # rᵖ power law

function (::FirstIntegral{PowerLaw{p},n})(X::Number) where {p,n}
    return p >= 0 ? X^p / (1 + n + p) : inv(X^(-p) * (1 + n + p))
end

Numerically computed integrals

In a general multi-physics BEM package, one might conceivably have a user-specified kernel KKK for which the first integral Kn\mathcal{K}_nKn​ is not known analytically. Performing the integral numerically at runtime would be too expensive, however, especially since the integrand may have an integrable singularity.

Instead, we will perform the integral Kn\mathcal{K}_nKn​ at compile-time (or, at least, outside the innermost BEM loops) for various XXX and use these values to compute a Chebyshev interpolating polynomial C(ξ)C(\xi)C(ξ). This polynomial will then be used to generate an efficient Kn(X)\mathcal{K}_n(X)Kn​(X).

There are three tricky points:

  • Kn(X)\mathcal{K}_n(X)Kn​(X) will probably be singular as X→0X\to 0X→0, which means we can't fit it directly to a polynomial. (This is not a problem for how Kn\mathcal{K}_nKn​ is used, because Kn\mathcal{K}_nKn​ is always used for integration over domains that do not include X=0X=0X=0.) This will be dealt with by requiring the user to specify the degree ppp of the singularity as X→0X \to 0X→0: i.e. Kn(X)=O(Xp)\mathcal{K}_n(X) = O(X^p)Kn​(X)=O(Xp) for X→0X\to 0X→0. We will factor out this singularity from Kn\mathcal{K}_nKn​ and fit the remaining (non-singular) function to a polynomial.

  • We need Kn(X)\mathcal{K}_n(X)Kn​(X) for X∈(0,∞)X \in (0,\infty)X∈(0,∞), whereas polynomial interpolation requires a finite interval, typically (−1,1)(-1,1)(−1,1). We will handle this by choosing a coordinate mapping ξ(X)∈(−1,1)\xi(X) \in (-1,1)ξ(X)∈(−1,1). Because such a coordinate mapping is necessarily singular, however, it will screw up the convergence of polynomial interpolation if we choose the wrong degree of singularity — we want a mapping such that Kn(ξ(X))\mathcal{K}_n(\xi(X))Kn​(ξ(X)) is nonsingular (e.g. a low-degree polynomial in ξ\xiξ) as X→∞X\to\inftyX→∞. To choose this, the user will specify a degree qqq of the decay rate as X→∞X\to\inftyX→∞, i.e. Kn(X)=O(Xq)\mathcal{K}_n(X) = O(X^q)Kn​(X)=O(Xq) for X→∞X\to \inftyX→∞.

  • XXX is dimensionful (it is a physical distance within one of the triangles or other geometric elements of the BEM basis). Mapping it to a dimensionless ξ\xiξ inevitably involves choosing a scale sss of XXX. This sss should be user-specified (e.g. it can be the median diameter of the BEM elements). That is, Kn(X)∼Xp\mathcal{K}_n(X) \sim X^pKn​(X)∼Xp for X≪sX \ll sX≪s and Kn(X)∼Xq\mathcal{K}_n(X) \sim X^qKn​(X)∼Xq for X≫sX \gg sX≫s.

In summary, the user will specify a PowerLawScaling{p,q,s} kernel type parameterized by p, q, and s. They will also define a (::PowerLawScaling{p,q,s})(r::Number) method that computes K(r)K(r)K(r).

The polynomial fit will be performed as follows, assuming p≤0p \le 0p≤0 and q≤0q \le 0q≤0. First, we let Ln(X)=Kn(X)/(sp+Xp)L_n(X) = \mathcal{K}_n(X) / (s^p + X^p)Ln​(X)=Kn​(X)/(sp+Xp), which eliminates the x→0x\to 0x→0 singularity while still having Ln∼XqL_n \sim X^qLn​∼Xq for X≫sX \gg sX≫s. Second, we let X=(1−ξ)1/q−21/qX = (1-\xi)^{1/q} - 2^{1/q}X=(1−ξ)1/q−21/q [or equivalently ξ=1−(x+21/q)q\xi = 1 - (x+2^{1/q})^qξ=1−(x+21/q)q], which maps ξ∈(−1,1)\xi \in (-1,1)ξ∈(−1,1) to X∈(0,∞)X \in (0,\infty)X∈(0,∞), and has the property that Xq≈1−ξX^q \approx 1-\xiXq≈1−ξ as ξ→1\xi\to 1ξ→1, so the coordinate remapping produces a nice degree-1 polynomial. Finally, we fit Ln(ξ(X))=C(ξ)L_n(\xi(X)) = C(\xi)Ln​(ξ(X))=C(ξ) to a Chebyshev polynomial CCC, and compute Kn(X)\mathcal{K}_n(X)Kn​(X) via Kn(X)=(sp+Xp)×Ln(ξ(X))\mathcal{K}_n(X) = (s^p + X^p) \times L_n(\xi(X))Kn​(X)=(sp+Xp)×Ln​(ξ(X)).

Chebyshev interpolation

The following routines compute the coefficients cnc_ncn​ of a Chebyshev interpolating polynomial C(x)=∑n=0N−1cnTn(x)C(x) = \sum_{n=0}^{N-1} c_n T_n(x)C(x)=∑n=0N−1​cn​Tn​(x) for a function f(x)f(x)f(x) on (−1,1)(-1,1)(−1,1), where Tn(x)=cos⁡(ncos⁡−1x)T_n(x) = \cos(n \cos^{-1}x)Tn​(x)=cos(ncos−1x) are the first-kind Chebyshev polynomials.

We compute these coefficients cnc_ncn​ by first evaluating f(x)f(x)f(x) at the Chebyshev points cos⁡(πn+1/2N)\cos\left(\pi\frac{n+1/2}{N}\right)cos(πNn+1/2​) for n=0,…,N−1n=0,\ldots,N-1n=0,…,N−1, for which the Chebyshev sum is equivalent to a type-III discrete cosine transform (DCT-III), so that the coefficients cnc_ncn​ are computed by a DCT-II. These are not quite the typical Chebyshev points, which correspond to a DCT-I: the difference is that the DCT-I corresponds to the closed interval [−1,1][-1,1][−1,1], i.e. it includes the endpoints, whereas our function may involve terms that blow up at the endpoints (although the overall function should be okay) so we don't want to evaluate it there.

We also provide a function evalcheb to evaluate C(x)C(x)C(x) for any x∈(−1,1)x\in(-1,1)x∈(−1,1) by a Clenshaw recurrence, and a macro version @evalcheb (analogous to Base.@horner) that generates a completely inlined version of this recurrence for the case where ccc is fixed.

Given a function fff and a tolerance, return enough Chebyshev coefficients to reconstruct fff to that tolerance on (−1,1)(-1,1)(−1,1)

julia> function chebcoef(f, tol=1e-13)
    N = 10
    local c
    while true
        x = chebx(N)
        c = chebcoef(float[f(y) for y in x])
        # look at last 3 coefs, since individual c's might be zero
        if maximum(abs,c[end:end-2]) < tol * maximum(abs,c)
            break
        end
        N *= 2
    end
    v₀ = maximum(abs,c) * tol
    return c[1:findlast(v -> abs(v) > tol, c)] # shrink to minimum length
end

Given chebchebcheb coefficients aaa, evaluate them for xxx in (−1,1)(-1,1)(−1,1) by Clenshaw recurrence

julia> function evalcheb(x, a)
    isempty(a) && throw(BoundsError())
    -1 ≤ x ≤ 1 || throw(DomainError())
    bₖ₊₁ = bₖ₊₂ = zero(x)
    for k = length(a):-1:2
        bₖ = a[k] + 2x*bₖ₊₁ - bₖ₊₂
        bₖ₊₂ = bₖ₊₁
        bₖ₊₁ = bₖ
    end
    return a[1] + x*bₖ₊₁ - bₖ₊₂
end

# inlined version of evalcheb given coefficents a, and x in (-1,1)
julia> macro evalcheb(x, a...)
    isempty(a) && throw(BoundsError())
    # Clenshaw recurrence, evaluated symbolically:
    bₖ₊₁ = bₖ₊₂ = 0
    for k = length(a):-1:2
        bₖ = esc(a[k])
        if bₖ₊₁ != 0
            bₖ = :(muladd(t2, $bₖ₊₁, $bₖ))
        end
        if bₖ₊₂ != 0
            bₖ = :($bₖ - $bₖ₊₂)
        end
        bₖ₊₂ = bₖ₊₁
        bₖ₊₁ = bₖ
    end
    ex = esc(a[1])
    if bₖ₊₁ != 0
        ex = :(muladd(t, $bₖ₊₁, $ex))
    end
    if bₖ₊₂ != 0
        ex = :($ex - $bₖ₊₂)
    end
    Expr(:block, :(t = $(esc(x))), :(t2 = 2t), ex)
end

Extract parameters from PowerLawScaling type

julia> pqsPowerLawScaling{p,q,s}(::PowerLawScaling{p,q,s}) = (p,q,s)

julia> @generated function (::FirstIntegral{P,n}, X::Real) where {P<:PowerLawScaling,n}
    # compute the Chebyshev coefficients (of the rescaled 𝒦ₙ as described above)
    K = P()
    p,q,s = pqsPowerLawScaling(K)
    
    𝒦ₙ = X -> quadgk(w -> w^n * K(w*X), 0,1, abstol=1e-12, reltol=1e-10)[1]
    Lₙ = p < 0 ? X -> 𝒦ₙ(X) / (s^p + X^p) : 𝒦ₙ # scale out X ≪ s singularity
    q > 0 && throw(DomainError()) # don't know how to deal with growing kernels
    qinv = 1/q
    c = chebcoef(ξ -> Lₙ((1-ξ)^qinv - 2^qinv), 1e-9)
    
    # return an expression that inlines the evaluation of 𝒦ₙ via C(ξ)
    quote
        X <= 0 && throw(DomainError())
        ξ = 1 - (X + $(2^qinv))^$q
        C = @evalcheb ξ $(c...)
        return $p < 0 ? C * (X^$p + $(s^p)) : C
    end
end

Notice that a Helmholtz kernel is also a power law kernel:

import GSL
exprel(n, x) = GSL. sf_exprel_n(n, x)
type Helmholtz{k} <: PowerLaw{-1, -1} end # exp(ikr) / 4 pi r
function(::FirstIntegral{Helmholtz{k}, n})(x) where {k,n}
	ikx = im * k * x
	return exp(ikx) * exprel(n, -ikx)/(4*pi*x)
end

# magnetic field integral equation
abstract type MFIE{k} <: PowerLaw{-3, -3} end # (ikr - 1) * exp(ikr) / 4pi r^3
function (::FirstIntegral{MFIE{k}, n})(x) where {k,n}
    ikx = im * k * x
    return exp(ikx) * (im * k * exprel(n - 1, -ikx)/((n - 1)*x) - exprel(n - 2, -ikx)/((n-2)*(x^2)) / (4*pi*x)
end
